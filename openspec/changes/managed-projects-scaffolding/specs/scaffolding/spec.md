## ADDED Requirements

### Requirement: mnmd init creates a managed project
`mnmd init` SHALL create a `monom.yaml` file and a `commands/` directory in the current working directory, resulting in a valid managed project.

#### Scenario: Init in empty directory
- **WHEN** user runs `mnmd init` in a directory with no existing monom project
- **THEN** the system creates `monom.yaml` with `commands: ./commands` and creates an empty `commands/` directory

#### Scenario: Init with language flag
- **WHEN** user runs `mnmd init --language python`
- **THEN** the generated `monom.yaml` includes `default_language: python`

#### Scenario: Init in directory that already has monom.yaml
- **WHEN** user runs `mnmd init` in a directory that already contains `monom.yaml`
- **THEN** the system exits with non-zero status and prints a message that a monom project already exists

#### Scenario: Init in directory that has executable monom (custom project)
- **WHEN** user runs `mnmd init` in a directory that already contains an executable `monom` file
- **THEN** the system exits with non-zero status and prints a message that a custom project already exists, suggesting the user remove it first if they want a managed project

### Requirement: mnmd init asks for language preference
When run interactively without a `--language` flag, `mnmd init` SHALL prompt the user to select a default language for command scripts.

#### Scenario: Interactive language selection
- **WHEN** user runs `mnmd init` without `--language` and stdin is a terminal
- **THEN** the system presents bash, python, and node as options and uses the selection as `default_language`

#### Scenario: Non-interactive defaults to bash
- **WHEN** user runs `mnmd init` without `--language` and stdin is not a terminal
- **THEN** the system defaults to bash without prompting

### Requirement: mnmd new command creates a command script
`mnmd new command <path>` SHALL create an executable script at the specified path under the project's `commands` directory.

#### Scenario: Create simple command
- **WHEN** user runs `mnmd new command deploy` in a managed project with `commands: ./commands`
- **THEN** the system creates `./commands/deploy` with appropriate shebang, executable permission, and minimal body

#### Scenario: Create nested command
- **WHEN** user runs `mnmd new command infra/provision`
- **THEN** the system creates `./commands/infra/provision`, creating the `infra/` directory if it doesn't exist

#### Scenario: Command already exists
- **WHEN** user runs `mnmd new command deploy` and `./commands/deploy` already exists
- **THEN** the system exits with non-zero status and prints a message that the command already exists

#### Scenario: Language from project default
- **WHEN** `monom.yaml` has `default_language: python` and user runs `mnmd new command deploy`
- **THEN** the generated script uses `#!/usr/bin/env python3` shebang

#### Scenario: Language flag overrides project default
- **WHEN** `monom.yaml` has `default_language: python` and user runs `mnmd new command deploy --language bash`
- **THEN** the generated script uses `#!/bin/bash` shebang (flag overrides YAML default)

### Requirement: Scaffolding generates transparent code
All code generated by scaffolding commands SHALL be minimal, readable, and self-explanatory. The generated code must not depend on monom internals or contain abstraction layers.

#### Scenario: Generated command script content (bash)
- **WHEN** a bash command script is generated
- **THEN** it contains only a shebang line (`#!/bin/bash`) and a minimal body (e.g., `echo "TODO: implement <command-name>"`)

#### Scenario: Generated command script content (python)
- **WHEN** a python command script is generated
- **THEN** it contains only a shebang line (`#!/usr/bin/env python3`) and a minimal body (e.g., `print("TODO: implement <command-name>")`)

#### Scenario: Generated command script content (node)
- **WHEN** a node command script is generated
- **THEN** it contains only a shebang line (`#!/usr/bin/env node`) and a minimal body (e.g., `console.log("TODO: implement <command-name>")`)

### Requirement: Scaffolding only works in managed projects
`mnmd new command` SHALL only operate in managed projects. In custom projects, it SHALL inform the user that scaffolding is unavailable.

#### Scenario: New command in custom project
- **WHEN** user runs `mnmd new command deploy` in a custom project (executable `monom` config, no `monom.yaml`)
- **THEN** the system exits with non-zero status and prints a message explaining that this is a custom project and scaffolding can't determine where to put commands

#### Scenario: New command outside any project
- **WHEN** user runs `mnmd new command deploy` in a directory that is not within any monom project
- **THEN** the system exits with non-zero status and prints a message that no monom project was found

### Requirement: Generated files have correct permissions
All command scripts generated by scaffolding SHALL have executable permission set.

#### Scenario: New command is immediately executable
- **WHEN** `mnmd new command deploy` creates a new script
- **THEN** the file has the executable bit set (chmod +x equivalent) and can be run directly

### Requirement: Scaffolding supports language options
Scaffolding commands SHALL support bash, python, and node as language choices for generated command scripts.

#### Scenario: Bash command generation
- **WHEN** language is bash (default or explicit)
- **THEN** shebang is `#!/bin/bash`

#### Scenario: Python command generation
- **WHEN** language is python
- **THEN** shebang is `#!/usr/bin/env python3`

#### Scenario: Node command generation
- **WHEN** language is node
- **THEN** shebang is `#!/usr/bin/env node`
